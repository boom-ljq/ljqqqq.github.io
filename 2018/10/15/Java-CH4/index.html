Java 继承与反射

Posted on 2018-10-15 Edited on 2020-02-02 In Coding

本章要点:

  • 子类可以继承或者覆盖父类的方法。
  • 使用关键字 super 来调用父类方法或者构造函数。
  • final 方法 不可能被覆盖，final 类 不能被继承。
  • abstract 方法 没有实现，abstract 类 不能被实例化。
  • 子类方法可以访问子类的受保护成员，但仅适用于同一个子类的对象。
  • 所有类都是 Object 类 的子类，它提供了 toString、equals、hasCode 和 clone 方法。
  • 所有枚举类型都是 Enum 类 的子类，它提供了 toString、valueOf 和 compareTo 方法。

Extend 继承

  • 子类可以继承或者覆盖父类的方法。
  • 使用关键字 super 来调用父类的方法或者构造函数。
  • final 方法 不能被覆盖，final 类 不能被继承。
  • abstract 方法 没有实现，abstract 类 不能被实例化。
  • 子类方法可以访问子类的受保护成员，但仅适用于同一个子类的对象。

父类与子类

关键字 extends 表示你正在从一个现有类派生出一个新类。这个已有类被称为父类，新类被称为子类。术语 父/子类 来自集合论。

定义和继承子类方法

这样一来，Manager 类 不仅可以使用 setBonus() 方法，也可以使用来自 Employee 类 的非私有方法了。

方法覆盖

有些时候，一个父类的方法在子类中需要修改。所以继承这个方法是不合适的，你需要覆盖这个方法。

不同于 this 引用，关键字 super 不是对象的引用，而是绕过动态查找方法并调用特定方法的指令。

在 覆盖 一个方法的时候，你必须仔细地匹配准确的参数类型。你也可以用 @Override 注解来标记这个新方法。

子类的构造

子类的构造函数无法访问父类的私有变量，因此它必须通过父类的构造函数来初始化它们。

父类的构造函数调用必须是子类的构造函数中的第一条语句。如果你忽略了调用父类构造函数，那么父类必须有一个没有参数的，被隐式调用的构造函数。

父类赋值

将子类对象赋值给父类对象是合法的。

Manager boss = new Manager(…);
Employee emp1 = boss;

在父类变量上调用方法时，虚拟机会查看变量的实际类型，并且定位方法的版本。这个过程被称为动态方法查找。

double salary = emp1.getSalary(); //调用的是Manager类的方法

父类赋值同样可以用于数组赋值，在技术上Java数组是协变的，但这也可能会导致错误的发生。

转换

使用 instanceof 操作符，可以判断父类引用是否可以转化为子类引用，这样在运行时就不会因为不兼容而导致报错。

final方法和类

当你将一个方法声明为 final 时，子类不可以覆盖它。

有时，你可能想要阻止其他人继承你的类。你需要在类的定义中使用 final 修饰符。例如，在Java API中的 URL 类、String 类和 LocalTime 类。

抽象方法和类

一个类可以定义没有实现的方法，强迫子类去实现方法。这样的方法以及其所属的类被称为抽象方法和抽象类，必须用 abstract 修饰符标记。通常会将非常通用的类设计成抽象类。

不同于接口，抽象类可以拥有实例变量和构造函数。抽象类也可以拥有非抽象方法。

构造抽象类的实例是不可能的。

Person p = new Person(“LIKAKI”); //错误

但是你可以用一个具体子类的对象去赋值.

//假设定义了一个Programmer类继承并实现了Person类的抽象方法
Person p = new Programmer(“LIKAKI”, 1234);

受保护访问

如果你想要限制方法只能被子类访问，或者想要子类可以访问父类的实例变量，你可以将父类方法或变量声明为 protected。在Java中，protected 赋予了包级别访问。

equals方法

Mehtod: boolean equals(Object other)
Object 类 的 equals() 方法用来测试一个对象是否被认为与另一个对象相等，方法所强调的是对象的引用，而非内容。

对于一些 基于状态的相等性测试 中，才有必要 覆盖 equals() 方法。
在定义 equals() 方法 时，有一些需要注意的常规步骤（以 Item 类为例，有 description 和 price 两个属性）：

  1. 检查两者的对象引用是否完全相同，这个检测耗费很小

    if (this == otherObject) return true;

  2. 与 null 做比较时，所有的 equals()方法 都要返回 false

    if (otherObject == null) return false;

  3. 由于 覆盖 了 Object 类 的 equals() 方法，需要把比较对象转换成实际的类型。在转换类型之前，需要使用 getClass() 方法

    if (getClass() != otherObject.getClass()) return false;

  4. 最后比较实例变量。对于基本类型，使用 == 操作符。但是对于 double类型，或者担心 NaN 或者无穷值，那么使用 Double.equals() 方法。对于对象，使用 Objects.equals()，一个 null 安全的 equals() 方法。

    Item other == (Item) otherObject;
    return Objects.equals(description, other.description)
    && price == other.price;

在子类中定义 equals() 方法时，先要调用父类的 equals() 方法。如果父类的检查没用通过，那么对象一定不相等。

hasCode方法

Mehtod: int hashCode()
如果你重新;定义了 equals() 方法，那么最好也重新定义 hashCode() 方法 以兼容 equals() 方法，不然当用户将你的类插入哈希集合或者HashMap时，它们可能会丢失.
Object.hash() 方法 是 空指针安全 的。

clone方法

Mehtod: protected Object clone()
clone() 方法 很复杂，也很少需要。除非你有一个很好的理由，不然不需要去 覆盖 这个方法。
clone() 方法 是用来创建一个”克隆”的对象，而不是复制对象的引用。即 拥有与原对象相同的状态的不同对象。
Object.clone() 方法 做了一个 浅拷贝。它简单的从原对象中拷贝所有的实例变量到被拷贝的对象中，当拷贝的实例变量都是基础变量或者不会改变，那没有问题。但如果不是，那么原对象和克隆对象将 共享可变的状态，这个时候就需要 覆盖 clone() 方法。

枚举

一个典型例子，去定义一个有四个实例的枚举类型。

public enum Size { SMALL, MEDIUM, LARGE, EXTRA_LARGE };

枚举方法

由于每一个枚举类型都有固定的实例集，因此可以直接用 == 来比较它们，也可以用 equals() 方法 来比较他们.
同样的，也不需要提供 toString() 方法，它会自动产生枚举对象的名称。就像上文例子中的 SAMLL MEDIUM 等。
与 toString() 方法 相反的操作是静态方法 valueOf()，它针对枚举类型而合成。

Size otherSize = Size.valueOf(“SMALL”);

你大概也可以直接使用

Size otherSize = Size.SMALL;

要获取一个包含所有该枚举类型实例的数组，可以使用 values() 方法.

Size[] allValues = Size.values();

所以，你可以在增强的for循环中使用这种方法来遍历一个枚举类型的所有实例:

for (Size s : Size.values()) { System.out.println(s); }

ordianl() 方法 返回实例在枚举声明中的位置，位置从0开始。

运行时类型信息和资源

在Java中，你可以在程序运行时查询一个对象属于那个类。此外，还可以查出类是如何加载的，并加载被称为 资源 的相关数据。

Class类

getClass() 方法返回一个被称为 Class 类 的对象。

Object obj = …;
Class c1 = obj.getClass(); // 这样的前缀不能省略

你可以通过静态方法 Class.forName() 来获取一个 Class对象.:

String className = “java.util.Scanner”;
Class<?> c1 = Class.forName(className); //一个描述java.util.Scanner类的对象

Class.forName() 主要用与构造那些可能在编译时还不被知晓的类的 Class对象。如果你已经提前知道需要哪一个类，就使用该类的名称:

Class<?> c1 = java.util.Scanner.class;

后缀 .class 也可以用来获取其他类型的信息:

Class c2 = String[].class; //描述数组类型String[] Class c3 = Runnable.class; //描述Runnable接口
Class c4 = int.class; //描述int类型 Class c5 = void.class; //描述void类型

Array 在Java中属于类，但是 接口、基本类型 和 void 都不是类。Class 这个名称有一些不太合适。

虚拟机为每种类型管理了一个唯一的 Class对象，因此你可以用 == 操作符来比较类对象。例如”

if (other.getClass() == Employee.class) …

类加载器

虚拟机指令存储在类文件。每个类文件都包含单个类或接口的指令。一个类文件可以存放在文件系统里，JAR文件里，远程某个位置或者甚至在内存中动态构建。一个类加载器负责加载字节流，并且在虚拟机中将他们转化成一个类或者接口。

bootstrap类加载器会加载Java类库(一般来自 /jre/lib/rt.jar文件)，它是虚拟机的一部分。
扩展类加载器 从 /jre/lib/ext 目录中加载”标准库扩展”部分。
系统类加载器加载应用程序类。它定位 classpath 中目录和JAR文件的类。
